# Garment Synthesis — 피팅샷 에디터 (로컬 도구)

모델 전신 사진 + 옷 사진을 넣으면 **그 모델이 그 옷을 입은 피팅샷**을 AI로 합성하는 로컬 테스트 도구입니다.
Node.js + TypeScript 단일 HTTP 서버(`editor-server.ts`, 포트 5177)로 돌아가며, 브라우저에서 접속해 씁니다.

버전(v2 → v2.4)이 한 서버 안에 공존하고, 각기 다른 경로로 접속합니다. 핵심 최신 버전은 **v2.4(`/new24`)** 입니다.

---

## 빠른 시작

```bash
npm install                       # 최초 1회
cp .env.example .env              # .env를 만들고 키를 채웁니다 (아래 참고)
npx ts-node editor-server.ts      # 서버 실행 (포트 5177)
```

실행 후 브라우저에서:

| 경로 | 버전 | 방식 |
|------|------|------|
| `/` | v2 | 메쉬 편집기(더미 목업) |
| `/new` | v2.1 | AI 드레이프(실제 모델 기준) |
| `/new2` | v2.2 | AI 드레이프(더미) + 최종 재합성 |
| `/new23` | v2.3 | 참조이미지 없이 **텍스트 핏 지시**로 합성 |
| **`/new24`** | **v2.4** | **cm 대신 "신체 착지점"으로 핏 지시 + 비율 잠금** |

> `.ts`를 고치면 서버 재시작이 필요합니다. `.html`/`.js`/`.txt`는 저장 후 브라우저 하드 새로고침(Ctrl+F5)만 하면 반영됩니다.

## v2.4 흐름 (핵심)

v2.4의 문제의식: **사진만으로는 옷의 절대 cm를 알 수 없고**, LLM에 raw cm를 던져 "핏을 계산해라" 하면 산수가 흔들려 "밑단이 어디 떨어지는지"가 들쭉날쭉했습니다. v2.4는 **계산은 코드가(결정론적), 문장은 LLM이** 담당하도록 역할을 분리했습니다.

### ① 모델 체형 분석
모델 전신 사진 → AI가 키·둘레·팔다리 길이 추정. 값은 직접 수정 가능하며, 이 값으로 **이 모델만의 "신체 세로 사다리"**(어깨·가슴·허리·골반·사타구니·무릎·발목 높이, 바닥=0)를 만듭니다.

### ② 옷 실측 분석 — 비율 잠금
옷 사진 → AI가 작업지시서식 실측(총장·어깨너비·가슴단면 등)을 추정하고 사진 위에 측정 위치를 화살표로 그립니다.

- **비율 잠금(자물쇠)**: 사진의 절대 크기는 몰라도 **사진 안의 픽셀 비율은 정확**합니다. 잠긴(🔒) 항목끼리는 픽셀 비율로 연동되어, 아는 실측 하나만 고치면 나머지가 함께 재계산됩니다("하나만 알면 전부 맞음"). 특정 값을 빼려면 그 항목을 🔓 풀면 됩니다(독립). 맨 위 자물쇠로 전체 잠금/풀림.
- **AI 추정 표시**: 아직 AI 추측 초기값인 항목엔 "AI 추정" 태그가 붙고, 직접 고치거나 연동 재계산되면 사라집니다.

### ③ 핏 / 기장 지시사항
- **코드(`fit-mapper.ts`)가 착지점을 계산** → 옷 총장/소매/단면을 신체 사다리 위 한 점에 매핑해 "밑단이 무릎 아래 종아리 상단 1/3 지점에 떨어진다", "소매가 손목을 3cm 지난 손등을 덮는다"처럼 **랜드마크 착지점**과 여유량 등급·실루엣을 확정합니다. 이 요약이 화면의 **FIT MAP**(읽기 전용)입니다.
- **선택한 핏 모델(LLM)이 그 FIT MAP을 문장화**만 합니다(재계산 금지). 그래서 산수 오류가 없습니다. 나온 문장은 항목별로 직접 편집 가능.
- **모델 선택**: Gemini 3.5 Flash / GPT-5.6 Sol / GPT-5.6 Terra / Claude Fable 5 / Claude Opus 4.8. (옷 분석 모델과 별개로 고름)

> **FIT MAP vs 아래 문장**: FIT MAP은 코드가 계산한 "정답지"(근거 숫자 포함, 읽기 전용), 아래 문장은 LLM이 그걸 옮긴 "답안"(편집 가능, 실제로 프롬프트에 들어감). 문장이 FIT MAP과 다른 위치를 말하면 LLM 실수이니 문장을 고치면 됩니다.

### ④ 피팅샷 생성
모델·옷 원본 + 편집된 지시사항을 최종 프롬프트(`prompt-24.txt`)에 넣어 합성. **생성 모델 복수 선택** 가능(각각 한 장씩 병렬 생성):

- Gemini(기본), GPT Image-2 (High/Medium), Seedream 4.5 / 5.0-lite, Qwen, OG 2K/4K
- 결과는 세로 높이를 통일해 나란히 놓고 핏 비교 가능. 여러 번 눌러 여러 장 생성.

**모드**: 상의/하의 모드(2벌) ↔ 원피스 모드(1벌, 원피스 또는 상하의 세트). **"핏 지시사항 없이 생성"** 체크 시 지시사항을 비워 순수 비교 가능.
