# Garment Synthesis — 피팅샷 에디터 (로컬 도구)

모델 전신 사진 + 옷 사진을 넣으면 **그 모델이 그 옷을 입은 피팅샷**을 AI로 합성하는 로컬 테스트 도구입니다.
Node.js + TypeScript 단일 HTTP 서버(`editor-server.ts`, 포트 5177)로 돌아가며, 브라우저에서 접속해 씁니다.

버전(v2 → v2.7)이 한 서버 안에 공존하고, 각기 다른 경로로 접속합니다. 핵심 최신 버전은 **v2.7(`/new27`)** 입니다.

> **v2.7 상세 명세**는 [`fitting-27-manual.md`](fitting-27-manual.md)를 참고하세요(전체 flow · 엔드포인트 · 모듈 · 프롬프트 · 데이터 구조).

## 핵심 설계 사상 (v2.4부터)

> **"계산은 코드가(결정론적), 문장은 LLM이."**

사진만으로는 옷의 절대 cm를 알 수 없고, LLM에 raw cm를 던져 "핏을 계산하라"고 하면 산수가 흔들려 밑단 착지 위치가 들쭉날쭉했습니다. 그래서 역할을 분리했습니다.

1. **코드(`fit-mapper.ts`)가** 옷 실측 ↔ 모델 실측을 결정론적으로 계산해 "밑단이 무릎 아래 종아리 상단 1/3에 떨어진다" 같은 **신체 랜드마크 착지점 + 여유 등급 + 실루엣**을 확정 → 이게 **FIT MAP**.
2. **LLM(핏 모델)은** 그 FIT MAP을 **문장화만** 합니다(재계산 금지).
3. **이미지 생성 모델**이 그 문장 + 옷 사진 + 모델 사진으로 최종 합성합니다.

> **FIT MAP vs 아래 문장**: FIT MAP은 코드가 계산한 "정답지"(근거 숫자 포함, 읽기 전용), 아래 문장은 LLM이 옮긴 "답안"(편집 가능, 실제 프롬프트에 들어감). 문장이 FIT MAP과 다르면 LLM 실수이니 문장을 고치면 됩니다.

## v2.7 흐름 (핵심)

v2.7 = **v2.6(옷 종류 자동판정 + 레이어링 + 액세서리/신발) + 체형 재렌더**. 화면은 4단계입니다.

### ① 모델 체형 분석 + (신규) 체형·헤어 재렌더
- 모델 전신 사진 → AI가 키·둘레·팔다리 실측 추정(값 수정 가능). 이 값으로 **이 모델만의 신체 세로 사다리**를 만듭니다.
- **체형 선택**: 남/여 각 15종 프리셋(키 대비 비율로 정의). 선택하면 어깨·가슴·허리·엉덩이·몸무게가 그 체형으로 바뀝니다.
- **헤어스타일/색상**: 여 18 + 남 12 프리셋, 색상 14종.
- **[모델 재렌더]**: 선택한 체형·헤어로 모델샷을 다시 생성(얼굴·키·포즈·배경 유지, 몸통 비율만 변경). 결과가 이후 단계의 모델 입력으로 교체되고, 재렌더 기록에서 이전 모델로 되돌릴 수 있습니다.

### ② 옷 실측 분석 — 종류 자동판정
- 상의·하의·원피스·세트를 **한 슬롯(최대 5장)에 아무거나** 올리면 AI가 종류를 스스로 판정(오판은 카드의 [종류] 드롭다운으로 교정).
- 같은 옷의 다른 각도 사진은 **자동 그룹핑**(대표 1장 + 뒤/옆 뷰). 드래그로 합치기/분리 가능.
- 같은 종류가 여러 장이면 **겹침(레이어) 순서**를 자동 정렬(이너→겉), 칩 드래그로 수동 조정.
- **비율 잠금(🔒)**: 사진의 절대 크기는 몰라도 픽셀 비율은 정확 → 잠긴 항목끼리 연동돼, 아는 실측 하나만 고치면 나머지가 재계산됩니다.

### ③ 핏 / 기장 지시사항
- **코드(`fit-mapper.ts`)가 착지점·여유·겹침·가림을 계산**(FIT MAP) → **핏 모델(LLM)이 문장화**만. 나온 문장은 항목별로 편집 가능.
- **모델 선택**: Gemini 3.5 Flash / GPT-5.6 Sol / GPT-5.6 Terra(기본) / Claude Fable 5 / Claude Opus 4.8.

### ④ 피팅샷 생성
- 모델·옷 원본 + 편집된 지시사항 + 옷 외형설명을 최종 프롬프트(`prompt-27*.txt`)에 넣어 합성.
- **뷰 모드**: 앞면만 / 앞·뒤(1:1) / 앞·완전좌·뒤·완전우 4방향(16:9).
- **생성 엔진 복수 선택**: Gemini, GPT Image-2 (High/Medium × 2K/1K). 각각 한 장씩 병렬 생성해 나란히 비교.
- **"핏 지시사항 없이 생성"** 체크 시 지시사항을 비워 순수 비교.

## 알려진 한계

- 무릎·발목 등 미측정 지점은 인체 비례 상수 보간이라 ±2~3cm 오차가 있습니다(구간 표현이라 흡수됨).
- 하의 밑단은 **자연 허리 착용 가정**으로 계산합니다(밑위 값은 참고 표시만). 하이/로우라이즈 보정은 이후 과제.
- 모델 실측·옷 기장 추정은 **Gemini 고정**이라 `GEMINI_API_KEY`가 없으면 그 경로가 실패합니다.
- Claude(Fable/Opus) 모델은 계정 크레딧이 있어야 동작합니다.
- 재렌더는 **원본 대비 방향+강도 지시**가 결정적입니다. 재렌더 후 다시 재렌더하면 "직전 결과 대비" 변화가 계산되니, 원본 기준으로 바꾸려면 기록에서 원본으로 되돌린 뒤 진행하세요.
